## V19 bug修复 (22:36)

### Bug 1: 地点消失（关键修复）
**根因：** `_restoreLocationUnlocks()`依赖`G.storyProgress.chapterId`在STORY数组中查找索引。如果存档中是旧章节ID（如ch1_first_quest），新STORY找不到→currentIdx=-1→循环不执行→所有地点只剩whitecity。
**修复：**
- 添加`_OLD_CHAPTER_MAP`映射旧章节ID→新ID（6条映射）
- 添加`_resolveChapterId()`函数先查STORY再查映射表
- `_restoreLocationUnlocks()`中使用resolved ID
- 添加等级兜底逻辑：currentIdx仍<0时按玩家等级解锁对应区域地点
- `loadGame()`中迁移旧章节ID到新ID
- `CHAPTER_UNLOCKS['ch1_arrival']`从[]改为['lake']——新玩家一开始就需要能出城

### Bug 2: 任务进度不显示
**根因：** sceneGuild和_showQuestsFromExplore中只有`aq.progressCount`存在时才显示进度（"进度:X/Y"），没有progressCount的单步任务完全不显示进度。
**修复：**
- 默认progressCount=1，所有任务都显示"进度:X/Y"格式
- `_showQuestsFromExplore`改用`_findQuestById`搜索所有任务类型（QUESTS+GUILD+SIDE+CHAIN），而非只搜QUESTS
- 已完成任务显示✅标记
- 城镇banner已有进度数字显示
