## 小地图黑屏hotfix（09:57）

**Bug**：修复#5时改了SVG viewBox做视口跟随，导致小地图一片黑。

**根因**：小地图原本用CSS transform（`centerMapOnPlayer()`）做视口跟随——`mapLayer`绝对定位+translate偏移。我加的SVG viewBox裁切与这个机制冲突：
1. viewBox裁切把地图内容裁出了可见区域
2. `centerMapOnPlayer()`计算用`cellTotal=54`（正确值），但viewBox里用了`cellSize=36+gap=2=38`，尺寸不匹配

**修复**：回退viewBox改动，恢复全图SVG渲染，让原有的CSS transform机制做居中。

### 关键教训
- **小地图滚动架构**：SVG全图渲染 + `#mapViewport`(overflow:hidden) + `#mapLayer`(position:absolute, will-change:transform) + `centerMapOnPlayer()`用CSS translate偏移
- **不要用viewBox做视口跟随**——和已有的CSS transform机制冲突
- `cellTotal=54`（cellSize=52+gap=2）是地图网格的正确单元尺寸

## mm-layout溢出修复（10:39-10:44）

### Bug：从小地图导航栏进入子场景后页面不能上划
- **根因**：从小地图导航栏进入"状态/背包/装备/任务/存档"时，`body.mm-layout`没被移除
- `mm-layout`下`#game { overflow:hidden; max-height:100dvh }`把内容锁死在一屏
- **修复**：在5个入口函数开头都加了`_leaveMiniMapLayout()`：
  - `_showStatusFromExplore`
  - `sceneInventoryFromExplore`
  - `sceneEquipmentFromExplore`
  - `_showQuestsFromExplore`
  - `sceneSaveLoad`
- 版本号推进到v=27c

### Bug：任务界面"返回探索"按钮太大
- 加了`max-width:160px; margin:8px auto 0; display:block`
- 和战斗"继续"按钮同规格

### 排查结论
- 小地图导航栏5个项目（状态/背包/装备/任务/存档）全部覆盖
- 小地图内部子视图（_minimapHelp、_minimapLegend、_minimapWeather等）保持mm-layout，因为它们返回renderMiniMap()会重新添加mm-layout，正确
- 城镇子场景（商店/铁匠/酒馆/公会等）不受影响，不从mm-layout进入
- scenePlayerInfo和sceneEquipment的返回目标如果是renderMiniMap()，会正确重新添加mm-layout
