# Bug修复记录 - 2026-06-12

## 本次修复的Bug

### Bug #6: applyStoryChoice 遍历方式错误
- **位置**: `applyStoryChoice()` 函数
- **问题**: 使用 `for(var i=0; i<STORY_CHOICES.length; i++)` 遍历，但 STORY_CHOICES 是对象而非数组，`.length` 为 undefined，导致 requiresFavor 检查永远被跳过
- **影响**: 5个依赖好感度的故事选择条件形同虚设（innkeep_white, smithkeep_ice, shadow_whisperer, captain_storm）
- **修复**: 改为 `for (var _gk in STORY_CHOICES)` 对象遍历

### Bug #7: generateRandomDrop 函数签名不匹配
- **位置**: `generateRandomDrop()` 函数 + 深渊潜行3处调用
- **问题**: 函数签名 `(isBoss)` 只接受布尔值，但深渊潜行中传入 `(levelScale, rarity)` — 数字+字符串
- **影响**: 深渊中所有掉落使用Boss级概率表（数字是truthy），且稀有度覆盖完全无效
- **修复**: 
  1. 扩展签名为 `generateRandomDrop(isBoss, rarityOverride)`
  2. 添加 `if (rarityOverride && RARITY_TABLE[rarityOverride]) rarity = rarityOverride;`
  3. 修正3处调用：传入正确的 isBoss 布尔值 + rarityOverride

### Bug #8: startBattle 未定义
- **位置**: 第6933行（旅行遭遇战）
- **问题**: 调用 `startBattle()` 但函数名为 `startBattleWith()`
- **影响**: 旅行中遭遇敌人时，战斗无法启动（函数调用报错）
- **修复**: 改为 `startBattleWith(G._travelEncEnemy)`

### Bug #9: checkBattleEnd 未定义
- **位置**: 第14818行（物品使用击杀敌人）
- **问题**: 调用 `checkBattleEnd()` 但该函数从未定义
- **影响**: 使用伤害类物品击杀敌人后，战斗无法结束
- **修复**: 改为 `endBattle()`

### Bug #10: gainXP 未定义
- **位置**: ddOnBattleWin() 函数中
- **问题**: 调用 `gainXP(xpReward)` 但该函数从未定义
- **影响**: 深渊战斗胜利后不获得经验值
- **修复**: 改为 `G.xp += xpReward; while (G.xp >= calcNextXp(G.level)) { doLevelUp(); }`
