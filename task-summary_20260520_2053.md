# 龙途传说 v5 - 7项修复实施

## 目标
修复用户试玩v5后反馈的7个问题。

## 已完成的修复

### Fix #1: 地图扩大20-30倍
- **方案**: 从手动4-8个区域的`miniMap`改为程序化网格生成
- 坠星湖畔: 9×9=81格, 幽暗森林: 10×10=100格, 破碎堡垒: 11×11=121格, 暗影矿洞: 12×12=144格
- 新增`generateMap()`函数，使用seeded随机保证地图一致性
- 新增`ROOM_THEMES`数据，每个地点20种房间名+5种描述
- 从起点到首领保证有通路（随机行走算法）
- 迷雾系统：只显示已访问和相邻房间

### Fix #2: 小地图简化为纯2D网格
- 移除旧的方向箭头和节点图
- 改为纯上下左右移动（N/S/E/W）
- 3×3方向按钮布局：⬆️北 / ⬅️西 ● ➡️东 / ⬇️南
- 不可移动方向按钮自动disabled

### Fix #3: 小地图可点击
- 网格地图每个格子可直接点击移动
- 只允许点击相邻（上下左右）的已揭示格子
- 点击非相邻格子提示"只能移动到相邻格子"
- SVG渲染，每格约28px，emoji图标标识房间类型

### Fix #4: Boss重生逻辑
- 击败Boss后设置`G.bossDefeated[bossKey] = true`
- bossKey格式改为`locationId_row_col`
- 离开区域时(`leaveMiniMap()`)清除该地点所有bossDefeated标记
- 重新进入区域时boss刷新

### Fix #5: 升级通知+自动跳转属性分配
- `endBattle()`重写：检测升级后显示🎉升级界面
- 显示等级、连续升级数、经验/金币/属性点奖励
- 按钮"✨ 分配属性点"跳转`goToAttrAlloc()`
- 按钮"⏭️ 稍后再分配"跳转`skipAttrAlloc()`
- `confirmAttrAllocation()`确认后回到探索/世界地图
- 分配属性时满血满蓝

### Fix #6: 起源选择锁定
- 新增`G.gameStarted`标志
- 游戏开始后设为true
- 起源故事页和玩家信息页的"重新选择起源"按钮仅在`!G.gameStarted`时显示

### Fix #7: 视觉风格
- **延后处理**：用户将下次提供参考图
- 当前SVG已到极限，需AI生成图片方案

## 技术细节
- `LOCATIONS`数据从`miniMap.areas[]`重构为`mapConfig`（行列+概率）
- 旧函数全部替换：`renderMiniMapVisual`→`renderGridMap`, `clickMiniMapNode`→`clickGridCell`, `moveMiniMap`→`moveGrid`
- 新增CSS：`.move-grid`, `.move-btn`, `.move-center`, `.level-up-info`, `.mini-controls`
- 宝箱/事件收集后房间type变为'path'防止重复触发
- `_mapCache`缓存已生成地图避免重复生成
