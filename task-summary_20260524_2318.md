# 故事选择卡死Bug修复

## 问题
选择命运后界面卡死，无法做任何操作

## 根因
`selectStoryChoice(effect)` 存 `G.storyChoices[effect]`（如 `combat_bonus`），但 `showStoryChoice(chapterId)` 检查 `G.storyChoices[chapterId]`（如 `ch1_arrival`）。key不匹配 → 选择后 `showStory()` 再次调用 `showStoryChoice()` → 再次弹出选择 → 无限循环卡死。

## 修复
1. `selectStoryChoice` 里先存 `G.storyChoices[chapterId] = effect`，标记章节已选择
2. 选择后返回之前场景(`G._currentScene`)而非强制 `showStory()`
3. 选择回顾从STORY_CHOICES反查显示中文文本，不再显示内部effect key
