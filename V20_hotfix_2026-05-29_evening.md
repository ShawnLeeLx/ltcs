# 龙途传说 V20 热修复 - 任务系统与旅店卡死

## 时间
2026-05-29 22:34

## 用户反馈的两个问题

### 问题1：公会任务太多，一次性出现25个任务
- 根因：没有任务上限机制，所有QUESTS(7个) + SIDE_QUESTS(~15个) + GUILD_QUESTS(3-5个)全部一次性显示
- 玩家体验：任务太多无法关注，不知道自己在做什么

### 问题2：第一章主线要求住旅店，但玩家身无分文导致卡死
- 根因：ch1_arrival目标含rest_inn（需10金），初始金币50够用但玩家可能在之前花光
- 实际上初始50金+白城旅店10金通常够用，但缺乏兜底机制

## 修复方案

### 任务系统改造（6处修改）
1. **公会任务生成量降低**：3-5个 → 2-4个
2. **公会公告板上限**：最多显示5个公会任务（MAX_GUILD_BOARD=5），已接取的保留
3. **同时接取上限**：最多8个活跃任务（MAX_ACTIVE_QUESTS=8），超出时接取按钮灰掉+提示
4. **任务计数显示**：公会页面显示"当前任务: X/8"
5. **任务卡片视觉**：满任务时卡片变灰+显示"任务已满"替代"点击接取"
6. **接取校验**：acceptQuest()中硬性校验，超出8个直接toast拒绝

### 旅店系统兜底（4处修改）
1. **第一章免费首住**：ch1_arrival阶段，老板娘主动说"新来的？今晚免费住！"
2. **免费逻辑**：restAtInn()检测ch1_arrival+_ch1InnRestDone标记，免费一次
3. **慈悲收留**：任何城镇，如果金币不够+HP<30%+该城镇未收留过，出现"请求收留"按钮
4. **mercyRest()**：每个城镇限一次免费收留，HP/MP全恢复，触发rest_inn故事事件

## 设计思路
- 任务数量控制用的是"软上限+硬上限"：软上限是公告板显示量（5个），硬上限是接取量（8个）
- 旅店兜底用的是"剧情免费+系统慈悲"双层：第一章剧情免费（符合叙事），后续是系统慈悲（每个城镇一次）
- 不改变现有游戏机制的核心逻辑，只是加了约束和兜底
