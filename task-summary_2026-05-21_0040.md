# 装备系统大重构 — 7槽+前缀+套装

## 目标
将3槽简陋装备系统升级为完整的7槽+6品质前缀+随机词条+5套装系统

## 完成内容

### 1. 7槽位 (weapon/helmet/armor/gloves/boots/belt/accessory)
- equipment对象扩展为7个槽位
- 所有属性计算函数(calcMaxHp/calcAtk/calcDef/calcMagicAtk/calcCrit/calcDodge)遍历7槽累加
- 6个起源初始装备扩展到2-3件/起源
- equipItem用item.slot直接映射而非type推断

### 2. 6品质+前缀+词条系统
- 6品质: 普通白(1.0x)/精良绿(1.1x)/稀有蓝(1.25x)/史诗紫(1.45x)/传说橙(1.7x)/神话红(2.0x)
- 7属性前缀: 狂暴/坚韧/狡诈/智慧/幸运/迅捷/巨人
- 7效果前缀: 燃烧/冰霜/吸血/暴击/会心/反击/连击
- 11词条池: str/agi/int/cha/luk/atk/def/hp/mp/critRate/dodge
- 品质决定词条数和前缀数，数值60%~130%随机浮动

### 3. 5套装系统
- 火焰领主(火伤)/守护者(防御)/幸运商人(幸运)/影刃(暴击)/秘法贤者(魔法)
- 2/4/6件三档奖励，全部已实现
- 战斗中集成: 火焰加成/死亡豁免/暴击追加/背刺/MP减免

### 4. 战斗系统集成
- playerAttack: 火焰灼烧/吸血/致命一击/连击/背刺
- enemyTurn: 反击率/守护者死亡豁免
- 新属性全部在战斗计算中生效
